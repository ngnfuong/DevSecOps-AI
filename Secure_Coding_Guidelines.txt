TÀI LIỆU HƯỚNG DẪN LẬP TRÌNH AN TOÀN (SECURE CODING GUIDELINES)
Áp dụng cho mọi dự án Java Spring Boot.

Quy định 1: Quản lý tài nguyên Database (Khắc phục lỗi Java:S2095 - Resource Leak)
- Lập trình viên TUYỆT ĐỐI KHÔNG quản lý Connection thủ công bằng cách mở nhưng quên đóng trong khối finally.
- Tiêu chuẩn bắt buộc: Phải sử dụng cú pháp `try-with-resources` (từ Java 7 trở lên). Cú pháp này yêu cầu khai báo Connection, PreparedStatement và ResultSet bên trong dấu ngoặc đơn của khối try() để hệ thống tự động đóng tài nguyên.
- Không được làm thay đổi cấu trúc package hay tên class hiện tại khi sửa lỗi.

Quy định 2: Quản lý thông tin xác thực (Khắc phục lỗi Hardcoded Secrets)
- Không được phép lưu trữ mật khẩu (password), API keys, hay tokens dưới dạng plaintext trực tiếp trong tệp mã nguồn (.java).
- Tiêu chuẩn bắt buộc: Mật khẩu cơ sở dữ liệu phải được nạp thông qua biến môi trường sử dụng lệnh `System.getenv("DB_PASSWORD")`. 

Quy định 3: Chống tấn công SQL Injection
- Tuyệt đối không được nối chuỗi (string concatenation) trực tiếp khi viết câu lệnh SQL.
- Phải sử dụng `PreparedStatement` và gán tham số bằng các hàm `setString()`, `setInt()`.

Quy định 4: Chống tấn công lệnh hệ điều hành (OS Command Injection)
- Không truyền chuỗi đầu vào của người dùng trực tiếp vào `Runtime.getRuntime().exec()`.
- Phải dùng `ProcessBuilder`, và truyền tham số dưới dạng một mảng (array/list) các đối số riêng biệt (VD: `new ProcessBuilder("ping", "-c", "3", ipAddress)`). 
- Validate cẩn thận định dạng IP trước khi gọi lệnh.

Quy định 5: Quản lý File Upload an toàn
- Cần có hàm sanitize tên file (ví dụ: dùng `FilenameUtils.getName()`) để loại bỏ `../` (Path Traversal).
- Phải kiểm tra phần mở rộng file (đuôi file) thông qua Whitelist (chỉ cho phép .png, .jpg, .pdf...).

Quy định 6: Kiểm soát truy cập trực tiếp (IDOR)
- Khi truy vấn thông tin theo ID (như Order ID), luôn phải xác minh xem ID đó có thuộc quyền sở hữu của người dùng đang đăng nhập (`currentLoggedInUser`) hay không bằng cách thêm điều kiện: `AND user_id = ?` trong câu SQL.

Quy định 7: Phòng chống Server-Side Request Forgery (SSRF)
- Luôn kiểm tra URL do người dùng cung cấp có thuộc danh sách an toàn (Whitelist domains/IPs) hay không trước khi gọi kết nối. Không cho phép hostname nội bộ, localhost hoặc IP 169.254.169.254.

Quy định 8: An toàn khi Parse XML (XXE)
- Vô hiệu hóa External Entities khi cấu hình parser:
  `factory.setFeature("http://apache.org/xml/features/disallow-doctype-decl", true);`

Quy định 9: An toàn Mật mã cơ bản (JWT/Crypto)
- Tuyệt đối không dùng chuỗi bí mật (Secret Key) quá ngắn hoặc code cứng (hardcode). JWT Secret nên được lưu trữ trong Vault hoặc biến môi trường `System.getenv("JWT_SECRET")`.

Quy định 10: Ghi nhật ký hệ thống an toàn (Khắc phục lỗi Java:S106)
- Tuyệt đối không sử dụng `System.out.println` hoặc `System.err.println` để ghi thông điệp log hoặc thông tin gỡ lỗi (debug) ra console.
- Tiêu chuẩn bắt buộc: Phải dùng các thư viện ghi nhật ký tiêu chuẩn (như Log4j 2, SLF4J, Logback) thông qua logger đã khai báo để ghi log có kiểm soát (như `logger.info()`, `logger.error()`, `logger.warn()`).
- Ví dụ: Thay vì `System.out.println("Message");` phải dùng `logger.info("Message");`

PHAN II: QUY TAC BAO MAT DA NGON NGU (MULTI-LANGUAGE RULES)
===================================================================

--- PYTHON ---

Quy dinh PY-1: Chong SQL Injection (Python)
- KHONG noi chuoi truc tiep vao cau SQL.
- Phai dung parameterized query voi dau ? (sqlite3) hoac %s (psycopg2).
- Sai: cursor.execute("SELECT * FROM users WHERE name = '" + name + "'")
- Dung: cursor.execute("SELECT * FROM users WHERE name = ?", (name,))

Quy dinh PY-2: Chong OS Command Injection (Python)
- KHONG dung os.system() hoac subprocess.run(shell=True) voi input nguoi dung.
- Phai dung subprocess.run() voi danh sach doi so rieng biet va shell=False.
- Sai: os.system("ping -c 4 " + host)
- Dung: subprocess.run(["ping", "-c", "4", host], shell=False)

Quy dinh PY-3: Chong Insecure Deserialization (Python)
- TUYET DOI KHONG dung pickle.loads() hoac yaml.load() voi du lieu khong tin cay.
- Thay the pickle bang json.loads() cho du lieu thuan tuy.
- Neu phai dung YAML, bat buoc dung yaml.safe_load() thay vi yaml.load().

Quy dinh PY-4: Chong Path Traversal (Python)
- KHONG noi chuoi truc tiep vao duong dan file.
- Phai dung os.path.basename() de loai bo ky tu ../.
- Dung: safe_name = os.path.basename(filename); open(os.path.join("/var/data", safe_name))

Quy dinh PY-5: Chong eval() Injection (Python)
- TUYET DOI KHONG dung eval() hoac exec() voi dau vao tu nguoi dung.
- Thay the bang ast.literal_eval() neu chi can parse kieu du lieu Python.

Quy dinh PY-6: Thuat toan Bam Mat khau An toan (Python)
- KHONG dung hashlib.md5() hoac hashlib.sha1() de bam mat khau.
- Phai dung bcrypt: import bcrypt; bcrypt.hashpw(password.encode(), bcrypt.gensalt())

Quy dinh PY-7: Quan ly Bi mat (Python)
- KHONG hardcode API key, password trong ma nguon.
- Phai dung os.environ.get("SECRET_KEY") hoac thu vien python-dotenv.

--- C# / .NET ---

Quy dinh CS-1: Chong SQL Injection (C#)
- KHONG noi chuoi vao SqlCommand.
- Phai dung SqlParameter hoac Entity Framework voi LINQ.
- Sai: new SqlCommand("SELECT * FROM users WHERE name = '" + name + "'")
- Dung: cmd.Parameters.AddWithValue("@name", name); WHERE name = @name

Quy dinh CS-2: Chong OS Command Injection (C#)
- KHONG truyen input nguoi dung vao Process.StartInfo.Arguments dang chuoi ghep.
- Phai set tung argument rieng: proc.StartInfo.ArgumentList.Add(host)

Quy dinh CS-3: Thuat toan Bam Mat khau An toan (C#)
- KHONG dung MD5.Create() hoac SHA1 de bam mat khau nguoi dung.
- Phai dung BCrypt.Net hoac PBKDF2 (Rfc2898DeriveBytes) voi salt ngau nhien.

Quy dinh CS-4: Random An toan (C#)
- KHONG dung System.Random de tao token, OTP, session ID.
- Phai dung System.Security.Cryptography.RandomNumberGenerator.

Quy dinh CS-5: Quan ly Bi mat (C#)
- KHONG hardcode connection string trong source code.
- Phai dung Environment.GetEnvironmentVariable() hoac ASP.NET Core IConfiguration.

--- JAVASCRIPT / NODE.JS ---

Quy dinh JS-1: Chong SQL Injection (JavaScript)
- KHONG noi chuoi vao cau SQL.
- Phai dung parameterized query: db.query("SELECT * FROM users WHERE name = ?", [name])

Quy dinh JS-2: Chong OS Command Injection (JavaScript)
- KHONG dung exec() voi chuoi noi input nguoi dung.
- Phai dung execFile() hoac spawn() voi mang doi so rieng biet.
- Sai: exec('ping -c 4 ' + host)
- Dung: execFile('ping', ['-c', '4', host])

Quy dinh JS-3: Chong XSS (JavaScript)
- KHONG dung innerHTML hoac document.write() de render du lieu tu nguoi dung.
- Phai dung textContent hoac thu vien DOMPurify de sanitize.

Quy dinh JS-4: Chong eval() Injection (JavaScript)
- TUYET DOI KHONG dung eval() voi dau vao tu nguoi dung.
- Thay the bang JSON.parse() cho viec parse du lieu JSON.

Quy dinh JS-5: Bam Mat khau An toan (Node.js)
- KHONG dung crypto.createHash('md5') de bam mat khau.
- Phai dung bcrypt: const hash = await bcrypt.hash(password, 12)

--- GO ---

Quy dinh GO-1: Chong SQL Injection (Go)
- KHONG noi chuoi vao cau SQL.
- Phai dung placeholder ? voi db.Query("SELECT * FROM users WHERE name = ?", name)

Quy dinh GO-2: Chong OS Command Injection (Go)
- KHONG dung exec.Command("sh", "-c", "cmd " + input).
- Phai truyen doi so rieng: exec.Command("ping", "-c", "4", host)

Quy dinh GO-3: Bam Mat khau An toan (Go)
- KHONG dung md5.New() de bam mat khau.
- Phai dung golang.org/x/crypto/bcrypt: bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)

Quy dinh GO-4: Random An toan (Go)
- KHONG dung math/rand de tao token bao mat.
- Phai dung crypto/rand: rand.Read(token)

--- PHP ---

Quy dinh PHP-1: Chong SQL Injection (PHP)
- KHONG dung mysqli_query() voi chuoi noi.
- Phai dung PDO voi prepared statement.
- Dung: $stmt = $pdo->prepare("SELECT * FROM users WHERE name = ?"); $stmt->execute([$name]);

Quy dinh PHP-2: Chong OS Command Injection (PHP)
- KHONG dung shell_exec() hoac system() voi input nguoi dung.
- Neu bat buoc: shell_exec("ping -c 4 " . escapeshellarg($host))

Quy dinh PHP-3: Chong XSS (PHP)
- KHONG echo truc tiep bien tu $_GET hoac $_POST.
- Phai dung htmlspecialchars($input, ENT_QUOTES, 'UTF-8') khi output ra HTML.

Quy dinh PHP-4: Chong Insecure Deserialization (PHP)
- KHONG dung unserialize() voi du lieu tu nguoi dung.
- Phai dung json_decode() thay the.

Quy dinh PHP-5: Bam Mat khau An toan (PHP)
- KHONG dung md5() hoac sha1() de bam mat khau.
- Phai dung password_hash($password, PASSWORD_BCRYPT) va password_verify().

--- C / C++ ---

Quy dinh C-1: Chong Buffer Overflow (C/C++)
- KHONG dung gets(), strcpy(), sprintf() khong gioi han kich thuoc.
- Phai dung fgets(buf, sizeof(buf), stdin), strncpy(), snprintf().

Quy dinh C-2: Chong Format String Vulnerability (C)
- KHONG dung printf(user_input) truc tiep.
- Phai dung printf("%s", user_input) hoac puts(user_input).

Quy dinh C-3: Chong OS Command Injection (C/C++)
- KHONG dung system() voi chuoi noi input nguoi dung.
- Phai dung execvp() voi mang doi so: char *args[] = {"ping", "-c", "4", host, NULL}; execvp("ping", args);

Quy dinh C-4: Quan ly Bo nho (C/C++)
- Moi malloc()/new phai co free()/delete tuong ung.
- Phai kiem tra NULL sau khi malloc().
- C++: Uu tien dung smart pointer (unique_ptr, shared_ptr) thay vi raw pointer.

--- RUBY ---

Quy dinh RB-1: Chong SQL Injection (Ruby)
- KHONG noi chuoi vao cau SQL hoac dung string interpolation truc tiep.
- Sai: db.execute("SELECT * FROM users WHERE name = '#{name}'")
- Dung: db.execute("SELECT * FROM users WHERE name = ?", [name])

Quy dinh RB-2: Chong OS Command Injection (Ruby)
- KHONG dung backtick hoac system() voi string interpolation.
- Phai dung: system("ping", "-c", "4", host) voi doi so rieng biet.

Quy dinh RB-3: Chong eval() Injection (Ruby)
- TUYET DOI KHONG dung eval(user_input).
- Thay the bang cac ham tinh toan an toan hoac parser rieng.

Quy dinh RB-4: Bam Mat khau An toan (Ruby)
- KHONG dung Digest::MD5.hexdigest() de bam mat khau.
- Phai dung bcrypt gem: BCrypt::Password.create(password, cost: 12)

PHAN III: QUY TAC BAO MAT NANG CAO (OWASP TOP 10 — 2021)
===================================================================

--- A04: INSECURE DESIGN ---

Quy dinh SEC-1: Chong Cross-Site Request Forgery (CSRF)
- Moi form HTML gui du lieu (POST/PUT/DELETE) PHAI co CSRF Token di kem.
- Java Spring: Su dung Spring Security CSRF Protection (mac dinh bat). KHONG tat csrf().disable() trong production.
- JavaScript: Gui CSRF token trong header X-CSRF-TOKEN khi goi API tu frontend.
- PHP: Su dung csrf_token() trong form hoac thu vien Symfony CSRF Component.

Quy dinh SEC-2: Chong Mass Assignment
- KHONG bind truc tiep toan bo request body vao entity/model ma khong gioi han truong.
- Java Spring: Dung @JsonIgnoreProperties hoac DTO (Data Transfer Object) de chi nhan cac truong cho phep.
- Python Django: Dung serializer voi fields = ['name', 'email'] thay vi fields = '__all__'.
- Node.js: Pick chi cac truong can thiet: const { name, email } = req.body;

--- A05: SECURITY MISCONFIGURATION ---

Quy dinh SEC-3: Cau hinh CORS an toan
- TUYET DOI KHONG dat Access-Control-Allow-Origin: * trong moi truong production.
- Phai chi dinh cu the cac domain duoc phep: .allowedOrigins("https://example.com")
- KHONG cho phep tat ca HTTP methods: chi allow GET, POST can thiet.

Quy dinh SEC-4: Cau hinh HTTP Security Headers
- Phai thiet lap cac header bao mat bat buoc:
  + X-Content-Type-Options: nosniff (chong MIME sniffing)
  + X-Frame-Options: DENY (chong Clickjacking)
  + Strict-Transport-Security: max-age=31536000 (bat buoc HTTPS)
  + Content-Security-Policy: default-src 'self' (chong XSS)
- Java Spring: Dung Spring Security .headers().frameOptions().deny()

Quy dinh SEC-5: Tat Debug Mode trong Production
- KHONG de debug=True (Django), SPRING_PROFILES_ACTIVE=dev (Spring Boot), hoac NODE_ENV=development khi deploy len production.
- Phai su dung bien moi truong de phan biet moi truong dev/staging/production.

--- A06: VULNERABLE AND OUTDATED COMPONENTS ---

Quy dinh SEC-6: Quan ly Dependencies An toan
- KHONG su dung thu vien (library) da biet co lo hong bao mat (CVE).
- Phai quet dependencies dinh ky bang cong cu: OWASP Dependency-Check (Java), npm audit (Node.js), pip-audit (Python), Trivy (container images).
- Phai cap nhat (update) cac thu vien co lo hong trong vong 72 gio sau khi phat hien.

--- A07: IDENTIFICATION AND AUTHENTICATION FAILURES ---

Quy dinh SEC-7: Quan ly Session An toan
- Session ID phai duoc tao bang ham ngau nhien an toan (SecureRandom, crypto/rand).
- Phai dat thuoc tinh HttpOnly, Secure, SameSite=Strict cho session cookie.
- Phai vo hieu hoa (invalidate) session sau khi nguoi dung dang xuat.

Quy dinh SEC-8: Xac thuc JWT An toan
- KHONG luu JWT token trong localStorage (de bi XSS danh cap).
- Nen luu JWT trong HttpOnly cookie hoac su dung co che Refresh Token.
- Phai kiem tra chu ky (signature), thoi gian het han (exp), va issuer (iss) cua JWT truoc khi chap nhan.
- KHONG dung thuat toan "none" hoac HS256 voi secret key ngan hon 256 bit.

Quy dinh SEC-9: Gioi han Ty le Dang nhap (Rate Limiting)
- Phai gioi han so lan dang nhap sai lien tiep (VD: toi da 5 lan trong 15 phut).
- Sau khi vuot nguong: khoa tam thoi tai khoan hoac yeu cau CAPTCHA.
- Java Spring: Su dung Bucket4j hoac Spring Security AuthenticationFailureHandler.

--- A09: SECURITY LOGGING AND MONITORING FAILURES ---

Quy dinh SEC-10: Khong Ghi Log Thong tin Nhay cam
- TUYET DOI KHONG ghi mat khau, so the tin dung, SSN, token vao file log.
- Sai: logger.info("User login with password: " + password)
- Dung: logger.info("User login attempt for: " + username) (chi ghi username, KHONG ghi password)
- Phai mask (che) du lieu nhay cam truoc khi ghi log: "card=****1234"

--- CHONG OPEN REDIRECT ---

Quy dinh SEC-11: Chong Open Redirect
- KHONG redirect nguoi dung den URL do ho cung cap ma khong kiem tra.
- Sai: response.sendRedirect(request.getParameter("url"))
- Phai kiem tra URL redirect thuoc danh sach whitelist hoac chi cho phep relative URL (bat dau bang "/").
